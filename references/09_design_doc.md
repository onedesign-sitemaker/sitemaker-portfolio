# 09_design_doc.md - デザイン解説書の作成ルール

初納品（_01）時にクライアントへ提供するデザイン解説書。サイトの各パーツが**なぜそのデザインになっているか**を、役割・意図・効果の観点から細かく説明する資料。

---

## 1. デザイン解説書の目的

- クライアントに「なんとなくオシャレ」ではなく「根拠のあるデザイン」であることを伝える
- 修正依頼時の共通言語を作る（「ヒーローのCTAボタンを…」等で指示できるようにする）
- デザインの価値を正しく理解してもらい、不要な「とりあえず変えてみて」を減らす
- SITEMAKERの専門性・品質の高さを示す

---

## 2. デザイン解説書の構成（必須セクション）

### セクション1: デザインコンセプト

戦略設計書の内容を踏まえ、デザインの方向性を説明する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎨 デザインコンセプト
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ デザインパターン: [Pattern X - パターン名]
  選定理由: [なぜこのパターンを選んだか。
  業種・ターゲット・戦略との関連]

■ 全体の印象
  [例: 「紺色×白の落ち着いた配色で、
  信頼感と清潔感を両立させたデザインです。
  リフォームを検討中の方が安心して相談できる
  雰囲気を目指しました。」]

■ デザインで解決していること
  - [戦略の柱1] → [デザインでどう表現したか]
  - [戦略の柱2] → [デザインでどう表現したか]
  - [戦略の柱3] → [デザインでどう表現したか]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション2: カラー設計

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎨 カラー設計
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ メインカラー: [色コード] [色名]
  使用箇所: ヘッダー、見出し、フッター
  意図: [例: 紺色は「信頼」「誠実」を連想させる色。
  建設業の堅実なイメージに合致]

■ ベースカラー: [色コード] [色名]
  使用箇所: 背景全体
  意図: [例: オフホワイトは目に優しく、
  長時間読んでも疲れにくい]

■ アクセントカラー: [色コード] [色名]
  使用箇所: ボタン、リンク、重要な数字
  意図: [例: オレンジは「行動」「活力」を促す色。
  問い合わせボタンに使うことで目を引く]

■ 配色バランス: 70:25:5
  ベース70% / メイン25% / アクセント5%
  → 画面がごちゃつかず、大事な箇所に目がいく設計
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション3: フォント設計

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 フォント（書体）設計
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 見出し用フォント: [フォント名]
  意図: [例: Noto Serif JP（明朝体）は
  格式・信頼感を演出。会社の歴史や実績を伝える
  見出しに最適]

■ 本文用フォント: [フォント名]
  意図: [例: Noto Sans JP（ゴシック体）は
  読みやすさ重視。スマホでも文字がつぶれない]

■ 文字サイズの設計
  - PC本文: 16px（標準的な読みやすいサイズ）
  - スマホ本文: 15px（片手で持って読める）
  - 見出し: 画面幅に応じて自動調整（小さすぎず大きすぎず）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション4: 各セクションの解説（ページごと）

ページの上から順に、各セクションの役割と設計意図を解説する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📄 トップページ各セクションの解説
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【1】ヘッダー（ページ上部の共通メニュー）
  ┌─────────────────────────────────┐
  │ ロゴ　　　　　メニュー　　電話番号 │
  └─────────────────────────────────┘
  役割: どのページにいても迷わないナビゲーション
  工夫: スクロールしても上部に固定表示。
       電話番号を右端に置き、すぐ電話できる設計

【2】ヒーロー（最初に目に入る大きなエリア）
  ┌─────────────────────────────────┐
  │　　　　　大きな写真　　　　　　　│
  │　　キャッチコピー　　　　　　　　│
  │　　[お問い合わせはこちら]ボタン　 │
  └─────────────────────────────────┘
  役割: 3秒で「何の会社か」「何が強みか」を伝える
  工夫: [具体的な工夫の説明]

【3】強み・特徴セクション
  役割: [説明]
  工夫: [説明]

  (...以降、全セクションを同様に記載)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション5: レスポンシブ（スマホ対応）の解説

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📱 スマホ表示について
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 対応方針
  パソコンとスマートフォンの両方で、見やすく・使いやすい
  デザインになるよう設計しています。

■ スマホでの主な変更点
  - メニュー → ハンバーガーメニュー（三本線アイコン）に
  - 2列レイアウト → 1列に縦積み
  - 文字サイズ → やや小さく調整（読みやすさは維持）
  - 電話番号 → タップで直接発信できるボタンに
  - 画像 → 画面幅いっぱいに表示

■ 御社のお客様の[○○%]がスマホからのアクセスと
  想定されるため、スマホでの見やすさを特に重視しています。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

```
デザインをベースにカスタマイズしています。

  → [なぜこのパーツを選んだか]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 3. 出力形式

1. **テキスト形式**でチャット上に表示（確認用）
2. 確認OKをもらったら、案件フォルダ内に `design-doc.md` として保存

ファイルパス: `client/[クライアント名]_01/design-doc.md`

※ 初納品フォルダ（_01）に保存する。_00（AI初稿）には不要。

---

## 4. 書き方のルール

- **クライアントはWEBの専門家ではない**前提で書く
- 「ヒーロー」「CTA」「BEM」等の専門用語は（）内で必ず説明するか、平易な言葉に置き換える
- 簡易なワイヤーフレーム（罫線で図示）を添えて視覚的に分かりやすくする
- 「なぜそうしたか」の理由を必ず添える。理由なしの説明は禁止
- 「かっこいいから」「流行っているから」は理由にならない。必ずビジネス上の根拠と紐づける

---

## 5. デザイン解説書を作成するタイミング

1. AI初稿（_00）が完成
2. 社内修正フロー（_00 → _01）の実施
3. **_01 完成時に、デザイン解説書を自動生成**
4. _01のフォルダ内に `design-doc.md` を保存
5. 「初納品物 + デザイン解説書」のセットでクライアントへ提供
