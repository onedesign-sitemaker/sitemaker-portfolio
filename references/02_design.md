# SITEMAKER デザイン・トンマナルール

AIはデザイン案およびコーディング指示を出力する際、本ドキュメントのルールを**例外なく遵守**すること。これはSITEMAKERの「絶対的なデザインの掟」である。

---

## 1. デザインの基本原則（共通ルール）

### 1-1. 余白とサイズの法則（8pxグリッド＋デザイン4原則）

- 要素のサイズや余白（Margin / Padding）は、すべて**「8の倍数」**（8, 16, 24, 32, 48, 64, 80, 120px...）で構成する。
- 視覚的なリズムを整えることを目的とする。
- **デザインの4原則（近接・整列・反復・対比）を徹底**すること。特に関連する要素同士は近づけ（margin: **8px〜16px**）、異なるグループ間は大胆に離す（margin: **64px〜120px**）ことで、情報のグループ化を明確にする。

### 1-2. カラーの黄金比

- **メインカラー**（ブランド色）**25%**
- **ベースカラー**（背景色）**70%**
- **アクセントカラー**（ボタン等）**5%**
- 上記比率を厳守し、画面が散らかるのを防ぐ。

### 1-3. コンテンツ幅と比率設計（3段階幅システム）

コンテンツ幅は**3段階**を使い分け、ページにリズムと開放感を生む。

| 段階 | CSS変数 | 値 | 用途 |
|------|---------|-----|------|
| **テキスト幅** | `--content-width` | **1200px** | テキスト主体のセクション、ナビゲーション |
| **ワイド幅** | `--content-width-wide` | **1400px** | 画像＋テキスト混在セクション、カードグリッド |
| **フルブリード** | `width: 100vw` | **画面全幅** | ヒーロー、CTA、画像帯、背景色セクション |

- **必須ルール**: 1ページにつき**最低2つのフルブリードセクション**を含めること（ヒーロー＋CTA等）。全セクションがmax-width内に収まる「箱詰め」レイアウトは**禁止**。
- フルブリードセクション内のテキストは `--content-width` で制約し、読みやすさを保つ。
- 画面分割（2カラム等）や要素のサイズ比率には、可能な限り**「黄金比（1 : 1.618）」**または**「白銀比（1 : 1.414）」**を意識した数値を設定し、視覚的な美しさを担保する。

### 1-4. レスポンシブ（スマホ表示）の厳格な数値ルール

- **ブレイクポイント**: PCとスマホの切り替えは **max-width: 768px** を基準とする。
- **スマホ時の余白縮小**: セクション間・要素内の余白はPC版の **50%〜2/3** に縮小し、スマホ画面に最適化する。
- **スマホ時のフォントサイズ**:
  - 本文は **14px〜15px** を最小とする。
  - 見出し（H2）はPC版の **70%〜80%** に縮小する。

### 1-5. アニメーションと状態変化（ホバー）の共通定義

- すべてのボタン・リンク・画像のホバー時には **transition: 0.3s ease**（または ease-in-out）を**必ず**適用する。
- 瞬時に色が変わるチープな実装は避ける。
- **推奨**: 不透明度を下げる（opacity: 0.7）、少し上に浮く（transform: translateY(-2px)）など、上品な挙動を基本とする。

### 1-6. 「絶対に使ってはいけない」NGデザイン

| 項目 | ルール |
|------|--------|
| **純黒の影の禁止** | 純黒（#000000）の濃いドロップシャドウは**絶対に使用しない**。影は **rgba(0, 0, 0, 0.05)〜0.1** 程度の薄く、ぼかしの大きい「モダンシャドウ」を使用する。 |
| **原色の禁止** | 赤（#FF0000）や青（#0000FF）などの原色は、**エラー表示を除き**デザインカラーとして絶対に使用しない。 |
| **絵文字（Unicode）の禁止** | 🏢・👷などのシステム絵文字をアイコンや装飾として使用することは、サイトがチープで素人っぽく見える最大の原因となるため、**例外なく絶対に使用しない**。 |

### 1-7. サイトマップと共通パーツの固定化（LP化の防止）

- コーポレートサイトは**1ページ完結のLP型ではなく**、「トップページ」「会社概要」「事業内容」などの**複数ページ構成**を前提とする。
- 全ページ共通の**ヘッダー（グローバルナビ）**と**フッター**はモジュールとして固定し、下層ページでも同一構造を引き継ぐ。

### 1-8. Webフォント（Google Fonts）の標準実装と業種別ルール

- 常に高品質なタイポグラフィを維持するため、**必ずGoogle Fonts等の無料WebフォントをCDN経由で読み込み**使用する。

| トーン | 推奨フォント |
|--------|--------------|
| **信頼・堅実** | Noto Sans JP / Noto Serif JP |
| **親しみ・温もり** | Zen Maru Gothic |
| **洗練・モダン** | 英語 Inter / 日本語 Noto Sans JP |
| **高級感・伝統** | Shippori Mincho / Noto Serif JP |

### 1-9. ファビコン（Favicon）とWebクリップアイコンの必須化

- `<head>` 内に**必ずファビコンの設定**を含める。
- 独自ロゴが未用意の場合でも、ブラウザのデフォルトアイコンになるのを防ぐため、**サイトのテーマカラーを用いたシンプルなSVG**や**企業名の頭文字**をダミーファビコンとして設定する。

### 1-10. タイポグラフィと文字組の美学（clamp必須）

**固定pxのfont-sizeは禁止。全てのfont-sizeに `clamp()` を使用すること。**

| 要素 | clamp() 最小値 | 推奨値 | 最大値 | 備考 |
|------|---------------|--------|--------|------|
| **h1** | 40px | 5vw | **72px** | ヒーロー見出し。大胆に大きく |
| **h2** | 28px | 3.5vw | **48px** | セクション見出し |
| **h3** | 20px | 2.5vw | **32px** | サブ見出し |
| **body** | 15px | 1.2vw | **18px** | 本文。14px以下は禁止 |
| **small/caption** | 12px | 1vw | **14px** | 注釈・キャプション |

- **ジャンプ率**: h1の最大値 / bodyの最小値 = **3.5倍以上**を必須とする。見出しと本文の差が小さいと全体がのっぺりする。
- **行間（line-height）**: 見出しは **1.2〜1.4**、本文は **1.7〜2.0** に設定し、窮屈さを排除する。
- **字間（letter-spacing）**: 本文は **0.05em〜0.08em** を基本とする。

### 1-11. セクション間余白（clamp必須）

**固定pxのセクション余白は禁止。全て `clamp()` を使用すること。**

| 要素 | clamp() | 備考 |
|------|---------|------|
| **セクション間** | `clamp(80px, 10vw, 160px)` | セクション同士の間隔 |
| **セクション内padding** | `clamp(40px, 5vw, 80px)` | セクション内の上下padding |
| **SP時セクション間** | `clamp(48px, 8vw, 80px)` | モバイル時は縮小 |

### 1-12. レイアウトの多様性ルール

- **禁止**: 全セクションが中央揃え1カラムのみのページ構成。単調で退屈な印象になる。
- **必須**: 1ページにつき**最低1つの非対称レイアウト**を含めること（例: テキスト40% / 画像60%の分割、broken grid、画像のブレイクアウト）。
- **推奨**: 画像はコンテンツ幅を突き破る配置を積極的に使う（ネガティブマージン、100vw、`calc(50% - 50vw)` テクニック）。
- **推奨**: スティッキーテキスト＋スクロール画像、グリッドのズラしなどモダンなレイアウト手法を取り入れる。

### 1-13. CTA（コンバージョン）UI設計

- ボタンは一目で**「押せる」と分かるデザイン**にすること。角丸（border-radius）、控えめなドロップシャドウ（box-shadow）、ホバー時の滑らかな変化（transition）を**必須**とする。
- スマホ操作を考慮し、タップ領域（min-height）は必ず **48px以上** を確保する。
- 右側に**矢印アイコン等を配置**してアクションを誘導する。

## 2. 高品質なUIコンポーネント・動的表現集

静的なHTMLだけでなく、以下の動的コンポーネント・リッチ表現を**積極的に採用**して実装すること。

### 2-1. ローディングアニメーション（オープニング）

- 初回読み込み時にシンプルなローディング画面（スピナーやロゴのフェードイン等）を表示する。
- 読み込み完了後にメインコンテンツがフワッと現れるオープニングアニメーションを**基本実装**とする。
- **条件**: 軽量かつ **1〜2秒程度**で完了する設計とする。

### 2-2. スライダー（カルーセル）

- 社員スナップ・施工実績などを複数見せる際は、静的な並びではなく **Swiper.js（CDN読み込み）** を使用したスライダーを実装する。
- 画像サイズ目安: **300px × 400px**
- スマホ: 1枚 / PC: 3〜4枚が収まる設計とする。

### 2-3. スクロール連動アニメーション

- 各セクション・画像が画面内に入った瞬間に、**下から上へフェードイン**するアニメーションを**標準実装**とする。
- **Intersection Observer API** 等を用い、**transform: translateY** と **opacity** の組み合わせで実装する。

### 2-4. 無限ループスライダー（Marquee）

- 取引先ロゴ一覧やイメージ写真の羅列には、**CSSの @keyframes** を用いた無限ループスライダーを取り入れる。

### 2-5. ヘッダー・フッターのリッチな装飾（あしらい）

- **ヘッダー（上部固定時）**:
  - 背景色 **rgba(255, 255, 255, 0.8)**
  - **backdrop-filter: blur(10px)** を適用した「グラスモーフィズム（すりガラス効果）」を推奨する。
- **フッター**:
  - コンテンツとの境界に、単なる直線ではなく**SVGを用いた「波線（Wave）」や「斜めの切り返し」**を配置し、視覚的なリズムを作る。

### 2-6. 見出し・リストのあしらい

- 見出し（H2, H3）やリストの先頭には、**CSSの疑似要素（::before / ::after）** を活用する。
- 「メインカラーの小さなドット」や「アクセントライン」を**必ず**配置し、のっぺりとしたテキスト表現を避ける。

### 2-7. アイコンの標準実装ルール

- **アイコンライブラリの必須使用**: **Google Material Icons** / **FontAwesome** / **Phosphor Icons** などのプロ向け無料WebアイコンライブラリをCDN経由で読み込み使用する。
- **SVGの直接記述**: オリジナルロゴ・フラットアイコンは、画像ファイルではなく**SVGコードをHTMLにインラインで直接記述**する。
- **ダミー画像**: 後から差し替える箇所には絵文字で代用せず、**https://placehold.co/400x300** などのダミー画像URLを配置する。

## 4. デザイン選定と独自性のルール

デザイン案を提案・実装する際、以下のルールを**厳守**すること。

### 4-1. 単一パターンの強制

- クライアントの要望から、最も適切なデザインパターン（**Pattern A〜I**）を**必ず1つだけ**厳選して適用する。
- 複数のパターンを混ぜたり、同時に複数提案することは**避ける**。

### 4-2. 独自性の注入

- 同じパターンでも、ヒアリングした**「企業独自の強み」**を分析し、最適な配色・レイアウトブロックを独自に組み合わせる。
- **量産型のコピペデザインにならない**よう構成する。

### 4-3. モダンな実装の強制

- レイアウト配置には**必ず Flexbox または CSS Grid** を使用する。
- クラス名は**BEM記法**などの規則に従い、**運用保守が容易な構造**とする。
