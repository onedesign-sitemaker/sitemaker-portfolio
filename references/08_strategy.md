# 08_strategy.md - WEB戦略設計書の作成ルール

クライアントの「現状」と「目指すゴール」を構造的に整理し、WEBサイトがそのゴール到達にどう貢献するかを設計・提案するための資料。**クライアントが読んで「なるほど、だからこのサイト構成なんだ」と納得できること**が最も重要。

---

## 1. 戦略設計書の目的

- クライアントに「なぜこのサイトを作るのか」を明確に伝える
- サイト制作の根拠を「戦略」として言語化する
- 「何を載せるか」「どういう構造にするか」の判断基準を共有する
- SITEMAKERの提案に説得力を持たせる

---

## 2. 戦略設計書の構成（必須セクション）

以下の順番で作成すること。各セクションは**クライアントが読む前提**で、専門用語を避け、分かりやすい言葉で書く。

### セクション1: 現状の整理

ヒアリングで得た情報をもとに、クライアントの現状を客観的にまとめる。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 現状の整理
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 事業概要
  [業種]、[所在地]、[従業員規模]、[創業年数]

■ 現在の集客チャネル
  - 紹介・口コミ: ○割
  - チラシ・広告: ○割
  - WEB経由: ○割（既存サイトあり/なし）

■ 強み（競合との差別化ポイント）
  1. [具体的な強み1]
  2. [具体的な強み2]
  3. [具体的な強み3]

■ 課題・悩み
  - [現状の課題を具体的に記述]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション2: ゴール設定

クライアントが目指すゴールを明確にし、WEBサイトがどう貢献するかを定義する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 ゴール設定
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ ビジネスゴール（最終的に達成したいこと）
  → [例: 月間問い合わせ件数を現在の5件から20件に増やす]

■ WEBサイトの役割（ゴール到達への貢献）
  → [例: 検索から流入した見込み客に「信頼できる会社だ」と
     感じてもらい、問い合わせに繋げる]

■ ターゲットユーザー
  - 誰: [年齢層、属性、地域]
  - どんな状態で見る: [スマホ/PC、検索ワード例]
  - 何を求めている: [情報ニーズ]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション3: WEB戦略（コアメッセージ）

ターゲットに対して、サイト全体で伝えるべきメッセージを定義する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 WEB戦略
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ コアメッセージ（サイト全体で伝えること）
  「[ターゲット]が[課題・不安]を感じたとき、
   [クライアント名]なら[具体的な理由]で安心できる」

■ 信頼構築の3つの柱
  1. [柱1: 例）実績の見える化 → 施工事例を写真付きで掲載]
  2. [柱2: 例）透明性 → 料金体系を明確に提示]
  3. [柱3: 例）人柄 → 代表メッセージとスタッフ紹介]

■ 行動喚起（ユーザーに取ってほしいアクション）
  → [例: 無料相談フォームから問い合わせ]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション4: サイト構成の根拠

サイトマップの各ページが「なぜ必要か」を戦略と紐づけて説明する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🗂 サイト構成と各ページの役割
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ トップページ（index.html）
  役割: 第一印象で信頼感を与え、各ページへ誘導する
  戦略との紐付け: コアメッセージをヒーローで伝え、
  3つの柱を視覚的に配置

■ 会社概要（about.html）
  役割: 「どんな会社か」を知ってもらい、信頼を深める
  戦略との紐付け: 柱3「人柄」→ 代表メッセージで想いを伝える

■ [各ページ同様に記載]

■ お問い合わせ（contact.html）
  役割: 行動喚起の受け皿。ハードルを下げる設計にする
  戦略との紐付け: 「無料」「簡単」「すぐ返信」を明示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### セクション5: 掲載すべきコンテンツの提案

各ページに「何を載せるべきか」を具体的に提案する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 掲載コンテンツ提案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ページ名]に載せるべきコンテンツ:

  ✅ [コンテンツ1]
     → 理由: [なぜこれが必要か、戦略との関連]

  ✅ [コンテンツ2]
     → 理由: [なぜこれが必要か]

  ❌ [載せない方がいいもの]
     → 理由: [なぜ不要か、逆効果になる理由]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## 3. 出力形式

戦略設計書は以下の形式で出力する：

1. **テキスト形式**でチャット上に表示（確認用）
2. 確認OKをもらったら、案件フォルダ内に `strategy.md` として保存

ファイルパス: `client/[クライアント名]_00/strategy.md`

---

## 4. 書き方のルール

- **クライアントが読む資料**であることを常に意識する
- 専門用語（SEO、CTA、CV等）は使わない。使う場合は必ず（）で説明を添える
- 「〜すべきです」ではなく「〜をおすすめします」のトーンで書く
- 根拠のない「効果があります」は禁止。必ず理由を添える
- 箇条書きと罫線を使い、視覚的に読みやすく整える

---

## 5. ヒアリング情報が不足している場合

戦略を設計するために以下の情報が不足していれば、追加でヒアリングする：

- 現在の集客方法と割合
- 競合他社（名前が分からなくても「近隣で同業の会社」程度でOK）
- 過去にWEBで困った経験
- 予算感・優先度（全ページ一気に作るか、段階的に作るか）

**ただし質問は1〜2個ずつ。一度に大量に聞かない。**
