# 05_test.md - 納品前品質チェックリスト

SITEMAKERで制作したサイトの**納品前**に、本チェックリストに沿って品質を検証すること。`dict/02_design.md` および `dict/04_coding.md` のルール準拠を確認の対象とする。**_04_完成データ** の納品前には全項目を実施すること。

---

## 1. コーディング・実装の検証

| # | 項目 | 確認内容 |
|---|------|----------|
| 1-1 | **BEM・クラス命名** | クラス名が BEM ベースでシンプルか。ブロック__要素--修飾子の関係が守られているか。 |
| 1-2 | **8pxグリッド** | 余白・サイズが 8 の倍数（8, 16, 24, 32, 48, 64…）で指定されているか。 |
| 1-3 | **jQuery 未使用** | jQuery に依存しておらず、Vanilla JS のみで実装されているか。 |
| 1-4 | **transition の適用** | ボタン・リンク・ホバー可能要素に `transition: 0.3s ease`（または同等）が付与されているか。チープな一瞬切り替えになっていないか。 |
| 1-5 | **レイアウト** | Flexbox または CSS Grid でレイアウトされており、float レイアウトに依存していないか。 |
| 1-6 | **スタイルの場所** | スタイルがクラスとCSSファイルで管理されており、IDやインラインスタイルで見た目を制御していないか。 |

## 2. レスポンシブ（768px 以下）の検証

| # | 項目 | 確認内容 |
|---|------|----------|
| 2-1 | **ブレイクポイント** | スマホ／PC の切り替えが **max-width: 768px** を基準に実装されているか。 |
| 2-2 | **余白の縮小** | **768px 以下**で、セクション間の余白（margin）および要素内の余白（padding）が PC 版の **50%〜2/3** に縮小されているか。 |
| 2-3 | **フォントサイズの縮小** | **768px 以下**で、本文が **14px〜15px 以上**を保ち、見出し（H2）が PC 版の **70%〜80%** に縮小されているか。 |
| 2-4 | **表示崩れ** | 768px 前後および 320px〜375px 幅で、横スクロール・はみ出し・重なりが発生していないか。 |

## 3. iOS / Safari 特有のバグ確認

| # | 項目 | 確認内容 |
|---|------|----------|
| 3-1 | **100vh のズレ防止** | フルスクリーンや固定高さでビューポート高さを使っている箇所で、**`100dvh`**（または同等の対策）が使われており、**`100vh` のみ**による iOS Safari のアドレスバー表示時のズレが起きていないか。実機またはシミュレータで確認する。 |
| 3-2 | **ホバー時のチラつき防止** | ホバーで `transform` や `opacity` が変わる要素に **`transform: translateZ(0)`**（または `will-change: transform` 等）が付与され、Safari でホバー時のチラつきが発生していないか。 |
| 3-3 | **iOS 実機確認** | iPhone（Safari）で表示崩れ・スクロールのカクつき・フォームのズレ等が発生していないか。 |

## 4. デザイン・トンマナの検証

| # | 項目 | 確認内容 |
|---|------|----------|
| 4-1 | **カラー比率** | メイン25%・ベース70%・アクセント5% に沿っているか。 |
| 4-2 | **NGデザイン** | 純黒の濃い影・原色・絵文字（Unicode）をデザインに使っていないか。 |
| 4-3 | **コンテンツ幅** | PC の max-width が 1000〜1080px または 1200〜1280px のいずれかに収まっているか。 |
| 4-4 | **単一パターン** | デザインパターン（Pattern A〜I）が1つに統一されているか。混在していないか。 |
| 4-5 | **Webフォント** | Google Fonts 等の Web フォントが適切に読み込まれ、指定トンマナと一致しているか。 |
| 4-6 | **ファビコン** | `<head>` 内にファビコン（またはダミーファビコン）が設定されているか。 |

---

## 5. アクセシビリティ・SEO の検証

| # | 項目 | 確認内容 |
|---|------|----------|
| 5-1 | **画像の alt** | すべての `<img>` に `alt` が付与されているか（装飾の場合は `alt=""`）。 |
| 5-2 | **見出し階層** | h1〜h6 が論理的な順序で使われ、飛び番号になっていないか。 |
| 5-3 | **外部リンク** | `target="_blank"` のリンクに `rel="noopener noreferrer"` が付いているか。 |
| 5-4 | **セマンティック** | header / nav / main / section / footer 等が適切に使われているか。 |

---

## 6. 動作・パフォーマンスの検証

| # | 項目 | 確認内容 |
|---|------|----------|
| 6-1 | **コンソールエラー** | ブラウザの開発者ツールで JS エラー・CSS エラーが出ていないか。 |
| 6-2 | **ローディング** | オープニング／ローディングアニメーションがある場合、1〜2秒程度で完了し、重くないか。 |
| 6-3 | **スライダー・カルーセル** | Swiper.js 等を使用している場合、スマホで1枚・PCで3〜4枚など指定どおりに動作しているか。 |
| 6-4 | **スクロール連動** | スクロール連動アニメーションが意図どおりに発火し、過度に重くないか。 |

---

## 7. 検証の実施タイミング

- **_01_社内修正** 以降の各フェーズで、変更範囲に応じて上記項目の該当部分をチェックする。
- **_04_完成データ** 納品前には、**全項目**（特に **§2 レスポンシブ** と **§3 iOS/Safari**）を必ず実施し、不具合・ルール違反を解消してから `backup/` へのコピーおよび資産化確認に進むこと。
